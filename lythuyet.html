BÀI LÀM LÝ THUYẾT

1. Position Absolute vs Relative

Relative: Phần tử vẫn nằm trong luồng tài liệu (document flow), việc di chuyển (top/left) chỉ là hình ảnh ảo so với vị trí gốc. Thường dùng làm mốc cho phần tử con.

Absolute: Bị nhấc khỏi luồng tài liệu, định vị theo phần tử cha gần nhất có position: relative. Nếu không có, nó định vị theo body.

Tình huống: Dùng Relative cho thẻ bao ngoài và Absolute cho Icon thông báo ở góc trên bên phải thẻ đó. Khi deploy, việc sử dụng các đơn vị như % hoặc vh/vw cùng position giúp layout co giãn tốt trên mọi thiết bị.

2. Domain, Hosting & DNS

Quy trình: Mua Domain (tên miền) -> Thuê Hosting (nơi chứa code) -> Trỏ bản ghi DNS (A hoặc CNAME) từ Domain về IP/Server của Hosting.

DNS: Đóng vai trò là "danh bạ", dịch tên miền (dễ nhớ) sang địa chỉ IP (máy tính hiểu).

Công cụ: GitHub Pages (miễn phí cho web tĩnh), Netlify (tự động build từ Git).

Rủi ro: Nếu cấu hình sai (ví dụ để lộ file .env hoặc quyền write công khai), hacker có thể chiếm quyền điều khiển hoặc đánh cắp dữ liệu người dùng.

3. Var, Let, Const & Hoisting

  • var: Function scope, cho phép redeclare, bị hoisting (có thể gọi trước khi khai báo).
  • let/const: Block scope (trong cặp {}), không cho phép redeclare. const không được gán lại giá trị.
// Lỗi thường gặp với var trong vòng lặp
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100); // In ra 3, 3, 3 thay vì 0, 1, 2
}

Debug: Sử dụng F12 (Chrome DevTools), tab "Sources" để đặt Breakpoint hoặc tab "Console" để kiểm tra lỗi đỏ.

4. Event Delegation

Thay vì gắn event listener cho từng thẻ li, ta gắn 1 listener duy nhất cho thẻ ul (cha). Nhờ sự kiện "nổi bọt" (bubbling), thẻ cha sẽ bắt được sự kiện của con.

Lợi ích: Tiết kiệm bộ nhớ, xử lý được cả các thẻ mới được thêm vào danh sách bằng JS sau này.
